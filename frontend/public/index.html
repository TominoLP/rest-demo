Was ist eine API?

Eine API (Application Programming Interface) ist eine definierte Schnittstelle, über die Programme miteinander kommunizieren können.

Vergleich: Ein Restaurant

  • Benutzer = du als Gast
  • Menü = API (zeigt was möglich ist)
  • Küche = Server (führt die Befehle aus)

SOAP – Der Vorgänger

SOAP (Simple Object Access Protocol) war viele Jahre der Standard für Webservices.

  • Pflicht: XML als Format
  • Streng, komplex, formal
  • WSDL für Schnittstellenbeschreibung
  • Gut geeignet für Banken/Unternehmen
  • Sehr sicher, aber schwerfälliger als REST

REST – Moderne APIs

REST (Representational State Transfer) nutzt die vorhandenen Möglichkeiten des Internets – vor allem HTTP-Verben und Statuscodes.

Aktionen (HTTP-Verbs)

GET

Lesen

Ressourcen abrufen.

POST

Neu erstellen

Neue Datensätze anlegen.

PUT

Aktualisieren

Bestehendes verändern.

DELETE

Löschen

Ressourcen entfernen.

HTTP-Statuscodes

Diese Codes zeigen den Erfolg oder Fehler einer Anfrage.

fetch()

fetch() ist das moderne Browser-API für HTTP-Anfragen. Es liefert immer ein Promise zurück. 

fetch('/items')
  .then(res => res.json())
  .then(data => console.log(data));
